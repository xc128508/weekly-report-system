# 周报系统响应式设计修改说明

## 一、项目类型

当前项目不是 Vue / React / Tailwind / Bootstrap 项目。

项目类型为：

Node.js + Express + server.js 内嵌 HTML + public 静态 CSS/JS

## 二、核心文件

### 1. src/server.js

页面 HTML 大概率都在这个文件里。

重点搜索关键词：

- layout
- sidebar
- topbar
- container
- card
- table
- form
- res.send

### 2. public/styles.css

全局主样式文件。

响应式设计建议优先在这里追加：

@media (max-width: 768px) {
  ...
}

### 3. public/admin-taskpool-template.css / public/admin-taskpool-template.js

管理员任务总表页面样式和交互。

对应页面：

/admin/task-pool

### 4. public/admin-taskpool-version-edit.css / public/admin-taskpool-version-edit.js

任务总表版本记录编辑区域。

### 5. public/intern-mobile-only.css / public/intern-mobile-only.js

实习生端手机适配历史文件。

需要判断是否保留，或者重新设计。

## 三、响应式设计建议

1. 不要优先用 JS 强制重排页面。
2. 优先通过 CSS 的 @media 实现响应式。
3. 保持 PC 端布局不变。
4. 以 768px 作为手机端断点。
5. 手机端主内容区宽度设为 100%。
6. 手机端表单改为单列布局。
7. 手机端 input、textarea、button 尽量 width: 100%。
8. 表格外层需要 overflow-x: auto，避免撑破手机屏幕。
9. 侧边栏手机端可以改成顶部导航或抽屉菜单，但要避免页面闪烁。
10. 修改前必须备份 src/server.js 和 public/styles.css。

## 四、不要修改的内容

不要改这些文件：

- data/db.json
- data/api-token.txt
- node_modules
- .env
- package-lock.json，除非确实新增依赖

## 五、建议优先改造页面

1. /intern/dashboard
2. /intern/daily-tasks/new
3. /admin/task-pool
4. /admin/dashboard
5. /boss/dashboard

## 六、验收标准

手机端需要满足：

1. 页面不横向撑破。
2. 顶部或侧边导航可正常使用。
3. 表格可以横向滑动。
4. 表单输入框足够大，方便点击。
5. 按钮不会挤在一起。
6. PC 端原布局不受影响。
